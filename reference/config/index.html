    Configuration Reference

    This page documents all configuration keys for Heta. Two files are involved:

    • config.yaml (project root) — API keys, database connections, and per-module model selections. Copy from config.example.yaml and fill in your credentials.
    • hetadb/config/db_config.yaml — Processing pipeline tuning parameters (batch sizes, worker counts, chunk settings).

    Domestic vs international models — Several sections list both an international default and a domestic (China-hosted) alternative in comments. Either works; choose based on your network environment and compliance requirements.

    config.yaml

    providers

    LLM and embedding API provider credentials. At least one provider must be configured.

    Key Description
    providers.dashscope.api_key Alibaba Cloud DashScope API key
    providers.dashscope.base_url DashScope OpenAI-compatible endpoint
    providers.siliconflow.api_key SiliconFlow API key
    providers.siliconflow.base_url SiliconFlow endpoint
    providers.openai.api_key OpenAI API key (optional)
    providers.gemini.api_key Google Gemini API key (optional)

    persistence

    External service connection strings. All three stores must be reachable for full functionality.

    PostgreSQL

    Key Default Description
    persistence.postgresql.host localhost Database host
    persistence.postgresql.port 5432 Database port
    persistence.postgresql.user — Username
    persistence.postgresql.password — Password
    persistence.postgresql.database — Database name

    Milvus

    Key Default Description
    persistence.milvus.host localhost Milvus host
    persistence.milvus.port 19530 Milvus gRPC port
    persistence.milvus.url — Alternative full URI (overrides host/port if set)

    Neo4j

    Key Default Description
    persistence.neo4j.url bolt://localhost:7687 Bolt connection URL
    persistence.neo4j.username neo4j Username
    persistence.neo4j.password — Password
    persistence.neo4j.database neo4j Database name

    hetadb

    Settings for the HetaDB document-processing and retrieval service.

    hetadb.fastapi

    Key Description
    hetadb.fastapi.host Bind host for the HetaDB FastAPI server
    hetadb.fastapi.port Bind port (default 8000)

    hetadb.workspace

    Key Description
    hetadb.workspace Absolute path to the directory where HetaDB stores knowledge base files

    hetadb.llm

    LLM used for graph extraction and answer synthesis.

    Key Example value Description
    hetadb.llm.model qwen3-32b Model name passed to the provider API
    hetadb.llm.provider dashscope Must match a key under providers

    hetadb.vlm

    Vision-language model used for image and PDF page understanding.

    Key Example value Description
    hetadb.vlm.model qwen2.5-vl VLM model name
    hetadb.vlm.provider dashscope Must match a key under providers

    hetadb.embedding_api

    Embedding model for vector indexing.

    Key Example value Description
    hetadb.embedding_api.model BAAI/bge-m3 Embedding model name
    hetadb.embedding_api.dim 1024 Embedding dimension; must match the Milvus collection schema
    hetadb.embedding_api.provider siliconflow Provider serving the embedding model

    hetadb.milvus

    Milvus collection settings for HetaDB.

    Key Description
    hetadb.milvus.collection_prefix Prefix prepended to all HetaDB Milvus collection names

    hetadb.query_defaults

    Default retrieval parameters applied when the caller does not specify them explicitly.

    Key Default Description
    hetadb.query_defaults.top_k 10 Vector candidate pool size per query
    hetadb.query_defaults.similarity_weight 1.5 Weight multiplier for vector similarity scores
    hetadb.query_defaults.occur_weight 1.0 Weight multiplier for occurrence-based (KG) scores

    hetamem

    Settings for the HetaMem episodic and knowledge-graph memory service.

    hetamem.memorykb

    MemoryKB is backed by LightRAG (NanoVectorDB + NetworkX). It uses lighter-weight models suitable for lower-latency personal memory.

    Key Example value Description
    hetamem.memorykb.llm.model gpt-4o-mini LLM for entity extraction and synthesis. Domestic alternative: qwen-turbo via DashScope.
    hetamem.memorykb.llm.provider openai Provider for the LLM
    hetamem.memorykb.embedding.model text-embedding-3-small Embedding model. Domestic alternative: text-embedding-v3 via DashScope.
    hetamem.memorykb.embedding.dim 1536 Embedding dimension
    hetamem.memorykb.embedding.provider openai Provider for the embedding model

    hetamem.memoryvg

    MemoryVG is backed by Milvus (vector store) and Neo4j (graph store), designed for structured fact extraction from multi-turn conversations.

    Key Example value Description
    hetamem.memoryvg.llm.model qwen3-32b LLM for fact extraction and memory reconciliation
    hetamem.memoryvg.llm.provider dashscope Provider for the LLM
    hetamem.memoryvg.embedder.model text-embedding-3-large Embedding model for semantic memory search
    hetamem.memoryvg.embedder.dim 1024 Embedding dimension
    hetamem.memoryvg.embedder.provider openai Provider for the embedder. Domestic alternative: text-embedding-v3 via DashScope.
    hetamem.memoryvg.vector_store.collection memoryvg Milvus collection name
    hetamem.memoryvg.graph_store (uses persistence.neo4j) Inherits Neo4j connection from the persistence block

    hetagen

    Settings for the HetaGen table and knowledge-tree generation service.

    Key Example value Description
    hetagen.llm.model gemini-3-flash-preview LLM for table schema and SQL generation
    hetagen.llm.provider gemini Provider for the LLM
    hetagen.vlm.model Qwen3-VL-32B VLM for visual table understanding
    hetagen.vlm.provider dashscope Provider for the VLM
    hetagen.embedding_api.model BAAI/bge-m3 Embedding model for retrieval
    hetagen.embedding_api.provider siliconflow Provider for the embedding model

    db_config.yaml

    Pipeline tuning parameters for HetaDB document processing. Located at src/hetadb/config/db_config.yaml. These values rarely need to change; adjust them when scaling to larger datasets or more powerful hardware.

    Top-level

    Key Default Description
    postgres_batch_size 500 Row batch size for bulk PostgreSQL inserts
    parse_max_workers 2 Maximum number of concurrent document parse tasks running at the same time

    parameter.chunk_config

    Controls text chunking during document preprocessing.

    Key Default Description
    chunk_size 1024 Target chunk size in tokens
    overlap 50 Token overlap between consecutive chunks
    max_workers 16 Thread pool size for parallel chunking

    parameter.graph_config

    Controls knowledge graph extraction from chunks.

    Key Default Description
    batch_size 2000 Number of chunks per graph extraction batch
    max_workers 200 Maximum concurrent LLM calls for entity/relation extraction

    parameter.graph_merge_config

    Controls deduplication and merging of extracted graph nodes and relations.

    Key Default Description
    parallel_batches 16 Number of merge batches to process in parallel
    batch_size 500 Entities per merge batch

    parameter.vector_config

    Controls embedding and vector index population.

    Key Default Description
    batch_size 2000 Chunks per embedding API call batch
    num_threads 8 Threads for concurrent Milvus upsert operations
